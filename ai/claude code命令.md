## 一、启动与CLI命令（在系统终端执行）

### 1、核心启动命令

| 命令                                           | 功能描述               | 典型场景                                                     |
| ---------------------------------------------- | ---------------------- | ------------------------------------------------------------ |
| `claude`                                       | 启动当前目录的交互模式 | 日常开发，进入 AI 结对编程状态。                             |
| `claude <目录路径>`                            | 启动并进入指定目录     | 例如 `claude /path/to/project`，直接打开特定项目。           |
| `claude -c` `claude --continue`                | 恢复最近一次会话       | 接着上次关闭前的对话继续，无需手动查找。                     |
| `claude --resume [会话名/ID]`                  | 恢复指定会话           | 如果不加参数，会弹出列表选择；加参数则直接恢复特定会话。     |
| `claude -p "提示词"` `claude --print "提示词"` | 一次性执行             | 发送问题 -> 获取回答 -> 立即退出。适合脚本自动化或快速提问。 |

### 2、配置与参数

| 参数                             | 功能描述             | 示例                                                         |
| -------------------------------- | -------------------- | ------------------------------------------------------------ |
| `--model <模型名>`               | 指定启动时使用的模型 | `claude --model opus` (强制使用 Opus) `claude --model haiku` (强制使用 Haiku) |
| `--add-dir <目录路径>`           | 添加额外的工作目录   | 让 AI 同时读取多个目录的代码（如前后端分离项目）。 `claude --add-dir ../shared-lib` |
| `--verbose`                      | 开启详细日志模式     | 启动后显示底层工具调用和调试信息，适合排查故障。             |
| `--dangerously-skip-permissions` | 跳过权限确认         | 高危命令。允许 AI 自动执行所有操作（包括写文件、运行命令）而不询问你。仅建议在受信任的脚本或沙箱环境中使用。 |
| `--version` `-v`                 | 查看版本号           | 检查当前安装的 Claude Code 版本。                            |
| `--help` `-h`                    | 查看帮助文档         | 显示终端命令的参数说明。                                     |

### 3、维护与管理命令

| 命令            | 功能描述            | 备注                                                         |
| --------------- | ------------------- | ------------------------------------------------------------ |
| `claude update` | 更新到最新版本      | 自动下载并安装最新的 Claude Code CLI。                       |
| `claude mcp`    | 进入 MCP 服务器配置 | 管理 Model Context Protocol 连接，用于连接外部数据源。       |
| `claude config` | 管理全局配置        | 通常用于高级设置，如 `claude config set -g language Chinese`（设置全局语言）。 |



## 二、斜杠命令（在claude界面内执行）

### 1、上下文与基于管理

| 命令       | 功能描述                                          | 典型场景与价值                                               |
| ---------- | ------------------------------------------------- | ------------------------------------------------------------ |
| `/init`    | 项目初始化。扫描当前目录，生成 `CLAUDE.md` 文件。 | 新项目必做。建立项目的“宪法”，让 AI 记住技术栈、目录结构和代码规范。 |
| `/memory`  | 编辑记忆。打开并修改 `CLAUDE.md` 文件。           | 当你想添加新规则（如“永远使用 TypeScript”）时，无需手动打开文件。 |
| `/compact` | 压缩上下文。总结对话历史，释放 Token 空间。       | 对话过长、AI 变慢或变贵时使用。保留核心逻辑，丢弃琐碎尝试。  |
| `/context` | 查看用量。可视化展示当前上下文窗口的 Token 占用。 | 判断是否需要运行 `/compact`，防止上下文溢出。                |
| `/clear`   | 硬重置。彻底清空当前会话历史。                    | 任务结束，准备开始一个完全不相关的新任务时。                 |

### 2、会话与状态控制

| 命令      | 功能描述                                       | 典型场景与价值                                               |
| --------- | ---------------------------------------------- | ------------------------------------------------------------ |
| `/resume` | 恢复会话。列出历史会话供选择恢复。             | 意外退出或切换分支后，找回之前的上下文。                     |
| `/fork`   | 创建分支。基于当前状态开启一个全新的平行会话。 | 尝试高风险重构时的“后悔药”，实验失败可随时切回主线。         |
| `/rewind` | 回退操作。撤销 AI 上一步的文件修改或决策。     | AI 刚刚执行了错误操作，需要立即撤销时。                      |
| `/rename` | 重命名会话。给当前会话起一个描述性的名字。     | 方便在 `/resume` 列表中快速识别特定任务（如“修复登录 Bug”）。 |

### 3、开发与工作流

| 命令               | 功能描述                                      | 典型场景与价值                                       |
| ------------------ | --------------------------------------------- | ---------------------------------------------------- |
| `/plan`            | 计划模式。AI 仅生成实施方案，不执行代码修改。 | 复杂任务前，先让 AI 出方案，你确认无误后再让它动手。 |
| `/todos`           | 任务列表。管理当前任务的进度清单。            | 跟踪复杂任务的执行进度，防止 AI 跑偏或遗漏步骤。     |
| `/diff`            | 查看变更。类似 `git diff`，展示代码修改差异。 | 提交代码前，快速审查 AI 到底改了哪些文件。           |
| `/review`          | 代码审查。AI 自动检查代码变更的 Bug 和规范。  | 提交前让 AI 充当审查员，查漏补缺。                   |
| `/security-review` | 安全审查。深度扫描代码中的安全漏洞。          | 检查 SQL 注入、XSS 等安全隐患，适合上线前自查。      |
| `/test`            | 运行测试。执行预设的测试命令。                | 快速验证新代码是否破坏了现有功能。                   |

### 4、系统配置与监控

| 命令      | 功能描述                                        | 典型场景与价值                                               |
| --------- | ----------------------------------------------- | ------------------------------------------------------------ |
| `/model`  | 切换模型。在 Opus/Sonnet/Haiku 之间切换。       | 攻坚难题切 `opus`；日常开发用 `sonnet`；省钱/简单任务用 `haiku`。 |
| `/cost`   | 查看费用。显示当前会话的 Token 消耗和预估金额。 | 实时监控预算，防止 Token 消耗过快。                          |
| `/config` | 打开配置。调整主题、权限、编辑器偏好等。        | 修改全局设置，如切换外部编辑器或调整权限模式。               |
| `/doctor` | 环境诊断。自动检查系统环境、网络和权限。        | 遇到连接错误或异常行为时，用于排查故障。                     |
| `/help`   | 帮助文档。显示所有可用命令的列表。              | 忘记命令用法时的救命稻草。                                   |

### 5、MCP与扩展

| 命令       | 功能描述                                       | 典型场景与价值                                      |
| ---------- | ---------------------------------------------- | --------------------------------------------------- |
| `/mcp`     | MCP 管理。配置 Model Context Protocol 服务器。 | 连接外部数据源（如 GitHub, Google Drive, 数据库）。 |
| `/add-dir` | 添加目录。将额外文件夹加入 AI 的工作视野。     | 前后端分离项目中，让 AI 同时读取两个仓库的代码。    |
| `/plugin`  | 插件管理。管理自定义插件。                     | 加载或卸载社区开发的扩展功能。                      |

### 6、隐藏与高级命令

| 命令             | 功能描述                                      | 适用场景                                             |
| ---------------- | --------------------------------------------- | ---------------------------------------------------- |
| `/btw <问题>`    | 旁注提问。向 AI 提问但不计入对话历史。        | 临时查询文档或概念，不想污染主上下文。               |
| `/effort <等级>` | 控制推理深度。设置 `low/medium/high`。        | 简单任务设为 `low` 以加快速度；复杂任务设为 `high`。 |
| `/batch <指令>`  | 批量处理。并行启动多个 Agent 处理大规模任务。 | 跨模块的框架迁移或批量 API 替换。                    |
| `/export`        | 导出会话。将当前对话导出为 Markdown 文件。    | 保存重要的设计讨论或会议记录。                       |
| `/copy [N]`      | 复制回复。将第 N 条回复复制到剪贴板。         | 快速获取 AI 生成的代码块，无需手动选择。             |



## 三、快捷键

### 1、通用控制快捷键

| 快捷键      | 功能描述  | 使用场景                                                     |
| ----------- | --------- | ------------------------------------------------------------ |
| `Ctrl + C`  | 中断/取消 | 立即停止 AI 当前的思考、生成或命令执行。                     |
| `Ctrl + D`  | 退出会话  | 结束并退出当前的 Claude Code 交互界面，等同于 `/exit`。      |
| `Ctrl + L`  | 清屏      | 清除终端屏幕上的所有内容，但保留对话历史，让界面恢复清爽。   |
| `Esc + Esc` | 撤销/回退 | 快速撤销 AI 对文件的上一次修改，或回退到上一个检查点。       |
| `Ctrl + R`  | 搜索历史  | 进入反向搜索模式，通过输入关键词快速查找并复用之前输入过的命令。 |
| `↑` / `↓`   | 浏览历史  | 在命令历史中上下翻页，快速调用之前输入过的完整指令。         |

### 2、输入与编辑快捷键

| 快捷键             | 功能描述     | 使用场景                                                    |
| ------------------ | ------------ | ----------------------------------------------------------- |
| `Option + Enter`   | 换行不发送   | 在输入多行指令或代码时换行，而不是立即发送给 AI。           |
| `Shift + Enter`    | 换行不发送   | 同上，在运行 `/terminal-setup` 后可用，是更通用的换行方式。 |
| `\` + `Enter`      | 换行不发送   | 同上，一种兼容性最好的换行方式，适用于所有终端。            |
| `Ctrl + A`         | 跳至行首     | 快速将光标移动到当前输入行的开头。                          |
| `Ctrl + E`         | 跳至行尾     | 快速将光标移动到当前输入行的末尾。                          |
| `Ctrl + U`         | 删除至行首   | 删除光标位置到行首的所有文本。                              |
| `Ctrl + K`         | 删除至行尾   | 删除光标位置到行尾的所有文本。                              |
| `Ctrl + W`         | 删除前一个词 | 删除光标前的一个单词，快速修正输入。                        |
| `Option + ←` / `→` | 按词移动光标 | 以单词为单位快速左右移动光标。                              |
| `Ctrl + V`         | 粘贴图片     | 将剪贴板中的图片直接粘贴到对话中，让 AI 分析图像。          |

### 3、AI交互与模式切换

| 快捷键        | 功能描述     | 使用场景                                                     |
| ------------- | ------------ | ------------------------------------------------------------ |
| `Shift + Tab` | 切换权限模式 | 循环切换 AI 的自动化级别：默认模式 → 自动接受编辑 → 计划模式等。 |
| `Tab`         | 切换思考模式 | 在“普通模式”（快速响应）和“深度思考模式”（更严谨的逻辑）之间切换。 |
| `Ctrl + O`    | 切换详细输出 | 展开或收起 AI 的工具调用详情，方便调试和查看其思考过程。     |
| `Ctrl + T`    | 切换任务列表 | 显示或隐藏由 `/todos` 命令管理的任务进度面板。               |
| `Option + P`  | 切换模型     | 快速在不同的 AI 模型（如 Sonnet, Opus, Haiku）之间切换。     |
